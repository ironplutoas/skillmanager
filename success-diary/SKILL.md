# 成功日记 Skill

专为嵌入式工程师和投资爱好者设计的每日成功记录工具，通过循序渐进的对话引导，5分钟内完成每日记录，同时支持历史日记的搜索、查询和统计分析功能。

## 核心原则
- 先易后难、贴合身份、对话式提问、自动汇总生成格式化日记
- 覆盖技术工作、投资实践、个人成长三大领域

## 默认配置
- **默认日记文件路径**：`H:\工作区\成功日记\成功日记.md`
- 可通过 `--path` 参数临时指定，或在对话中修改默认路径

## 使用方式

### 写日记模式
当用户调用 `/success-diary` 或 `/success-diary --path "xxx.md"` 时，进入写日记模式，按照以下流程引导用户：

1. **开场引导**
   输出："哈喽～ 现在我们用 5 分钟完成今天的成功日记！先配合回答几个小问题，贴合你的嵌入式工作和投资爱好，很简单。首先，今天的日期是？心情指数 1-10 分打几分呀？"

2. **嵌入式工作成功模块**
   - 第一问："今天有没有解决技术问题或完成项目模块？简单说一句。"
   - 追问（最多1次）："用了什么方法？或读了什么工科工具书？"
   - 后续追问（最多1次）："对你后续项目推进有什么帮助？"
   - 若用户表示无内容：引导填写替代内容，如"那说说有没有读技术相关的书籍或资料？"

3. **投资与经济成功模块**
   - 第一问："今天有没有做股票操作或读经济类书籍？"
   - 追问（最多1次）："遵循了什么原则？或看懂了哪个经济概念？"
   - 后续追问（最多1次）："这次操作让你规避了什么风险？或有什么新的感悟？"
   - 若用户表示无内容：引导填写替代内容，如"那说说有没有关注经济相关的新闻或知识？"

4. **个人成长成功模块**
   - 第一问："今天有没有读历史/全球经济书籍，或坚持某个习惯？"
   - 追问（最多1次）："给你带来了什么新的思考或具体变化？"
   - 若用户表示无内容：引导填写替代内容，如"那说说有没有什么生活上的小收获？"

5. **关键复盘模块**
   - 第一问："今天的核心收获，用 1 句话说说？"
   - 第二问："有什么可复用经验？"
   - 第三问："明天的关键目标，说 1-2 个就好？"

6. **汇总输出**
   将用户所有回答整理为以下格式化内容，自动追加到日记文件末尾：
   ```markdown
   ---
   date: YYYY-MM-DD
   ---

   ## 【嵌入式×投资成功日记】

   **日期：** YYYY年MM月DD日（星期X）  **心情指数：** N/10

   ### 一、嵌入式工作成功
   （整理后的用户回答，去除口语化冗余，保留核心表达）

   ### 二、投资与经济成功
   （整理后的用户回答）

   ### 三、个人成长成功
   （整理后的用户回答）

   ### 四、关键复盘
   1. **今日核心收获：** （一句话）
   2. **可复用经验：** （一句话）
   3. **明日关键目标：**
      - 嵌入式工作：（目标）
      - 投资学习：（目标）
   ```

### 回顾模式
当用户调用 `/success-diary review` 或带以下参数时，进入回顾模式：

| 参数 | 示例 | 说明 |
|------|------|------|
| `--recent N` | `--recent 7` | 查看最近 N 天摘要 |
| `--from YYYY-MM-DD --to YYYY-MM-DD` | `--from 2026-03-01 --to 2026-03-31` | 按日期范围查询 |
| `--search KEYWORD` | `--search STM32` | 按关键词搜索 |
| `--stats` | `--stats` | 统计分析 |
| `--path "xxx.md"` | `--path "我的日记.md"` | 指定日记文件路径 |

#### 功能实现说明：
1. **最近N天摘要（--recent）**
   - 读取日记文件，提取最近N条日记的日期、心情指数，以及每个模块的1句摘要
   - 输出为清晰的列表或表格形式，便于快速浏览

2. **日期范围查询（--from --to）**
   - 解析日期范围参数，筛选范围内的所有日记条目
   - 按日期从新到旧排序，输出完整内容或摘要（根据用户需求）

3. **关键词搜索（--search）**
   - 遍历所有日记条目，搜索包含指定关键词的段落
   - 输出匹配的内容及其对应的日期，高亮显示关键词

4. **统计分析（--stats）**
   - 分析所有日记数据，输出以下统计结果：
     - **心情趋势**：最近30天心情平均值及波动情况
     - **技术突破频率**：嵌入式模块中技术突破类记录的次数和频率
     - **投资操作统计**：投资操作次数、经济类书籍阅读数量统计
     - **习惯坚持情况**：各类习惯的坚持天数排名

## 对话规则
1. 每个模块最多追问1次关键问题，避免过多追问占用用户时间
2. 若用户表示某模块无内容，引导填写替代内容，不强制要求
3. 汇总时整理语言，去除口语化冗余词，保留核心表达
4. 单个模块提问不超过1分钟，总耗时控制在≤5分钟
5. 语气亲切、简洁，避免专业术语过多，贴合日常对话
6. 所有模式下，若用户未指定日记路径，优先使用默认路径，若默认路径文件不存在，自动创建
