# 股票交易复盘流程

AI主动诊断交易问题，识别冲突点，给出改进方案。

---

## 核心原则

**诊断原则**：
- **AI分析为主**：收集关键信息后，AI主动分析问题，而非让用户自己反思
- **识别冲突**：重点识别"理论规则 vs 实际操作"的冲突点
- **给出方案**：AI直接给出改进方案，用户只需确认或调整

**不要让用户做的事**：
- ❌ 问用户"做得好的地方是什么？做得不好的地方是什么？"
- ❌ 问用户"下次如何调整操作策略？"
- ❌ 让用户自己总结问题原因

**应该由AI做的事**：
- ✓ 收集基本信息后，分析操作是否符合预设规则
- ✓ 识别冲突点并指出："你判断逻辑成立，却因为XX原因卖出，这是规则冲突"
- ✓ 直接给出改进方案，征求用户确认

---

## 复盘流程

### 第一阶段：收集基本信息（快速收集，不要拖沓）

一次性收集以下信息，可以合并提问：

**必问信息**：
1. 股票名称和代码
2. 买入逻辑类型（景气度/红利/困境反转/事件驱动）
3. 买入逻辑具体描述
4. 买入时间、价格、仓位
5. 卖出时间、价格、原因
6. 持仓期间是否有多次买卖操作？（如有，简要说明）

**快速收集方式**：
> "请告诉我这次交易的基本情况：股票名称、买入逻辑类型（景气度/红利/困境反转/事件驱动）、买入时间和价格、卖出时间和价格、卖出原因。如果有多次买卖操作，也简要说明一下。"

---

### 第二阶段：AI诊断分析（核心环节）

收集完基本信息后，**立即进入诊断分析**，不要继续问用户。

#### 诊断步骤

**1. 匹配操作模板**

根据用户说的逻辑类型，匹配对应的操作规则：

- 景气度逻辑 → 检查是否按"逻辑止损"而非"成本线止损"
- 红利逻辑 → 检查是否因价格波动而非分红恶化卖出
- 困境反转逻辑 → 检查是否在反转验证中提前退出
- 事件驱动逻辑 → 检查是否因事件未兑现而非情绪卖出

**2. 识别规则冲突**

对比用户实际操作与预设规则，识别冲突点：

| 冲突类型 | 诊断标准 |
|----------|----------|
| **卖出冲突** | 用户说逻辑成立，却因价格/情绪卖出 → 规则冲突 |
| **买入冲突** | 用户在逻辑未验证时买入，或在高位追涨 → 规则冲突 |
| **仓位冲突** | 仓位过重导致心态不稳 → 需要调整仓位管理 |
| **时机冲突** | 入场时机在景气末尾或反转未确认 → 需要改进时机判断 |

**3. 分析问题根源**

识别表面问题后，分析深层原因：

| 表面问题 | 深层原因可能 |
|----------|--------------|
| 情绪卖出 | 仓位过重、浮盈回撤触发恐慌、没有心理缓冲 |
| 下跌途中买入误判 | 没有判断上行/下行转折的规则 |
| 后知后觉卖出 | 没有设置预警信号，跌到关键位置才反应 |
| 买入位置偏高 | 没有判断景气阶段的入场时机规则 |

**4. 给出改进方案**

直接给出具体可执行的改进建议，格式：

> **诊断结果**：
> 你这次交易的问题是XXX，原因是XXX。
>
> **改进方案**：
> 1. XXX调整
> 2. XXX调整
> 3. XXX调整
>
> 你觉得这些改进建议可以采纳吗？

## 操作策略模板

### 1. 景气度逻辑

| 阶段 | 操作规则 |
|------|----------|
| **买入** | 景气逻辑成立 + 财务安全验证 + 价格不在高位（判断景气阶段：初期/中期优先，末尾等回调） |
| **仓位** | 初始5%，逻辑验证后加仓至10%以内 |
| **加仓条件** | 逻辑验证点出现（涨价确认、订单放量）且股价未透支预期 |
| **减仓条件** | 景气拐点信号出现（价格见顶、库存累积、竞争者涌入） |
| **卖出触发** | ① 景气逻辑兑现完毕 ② 景气拐点确认 ③ 买入逻辑被证伪 |
| **止损规则** | 逻辑止损优先：逻辑证伪立即止损。心理止损兜底：买入价-5%（仅作缓冲，不触发主动卖出） |

**关键诊断点**：
- 如果用户说"逻辑成立"却因"跌到成本线"卖出 → **规则冲突**，原因是"不亏本"情绪底线压倒理性判断
- 如果用户在"下跌途中"买入 → 需要判断是"回调"还是"反转"，用"价格+逻辑"双重验证

**改进方案模板**：
| 问题 | 改进建议 |
|------|----------|
| 情绪卖出（逻辑成立却因价格卖出） | 1. 设置心理止损线（买入价-5%）替代成本线<br>2. 降低初始仓位至5%减轻心态压力<br>3. 每周持仓自检："逻辑是否成立？" |
| 下跌途中买入误判 | 1. 判断上行/下行：跌破10周线+逻辑松动→反转，跌破10周线+逻辑成立→回调<br>2. 下行周期等待企稳+逻辑恢复再入场 |
| 买入位置偏高 | 1. 入场前判断景气阶段<br>2. 景气末尾（价格已翻倍）等待回调后入场 |
| 后知后觉卖出 | 1. 设置预警信号（如跌破5周线触发审视）<br>2. 逻辑松动时主动减仓，而非跌到关键位置才反应 |

---

### 2. 红利逻辑

| 阶段 | 操作规则 |
|------|----------|
| **买入** | 分红稳定 + 股息率达标 + 财务安全 + 估值合理 |
| **仓位** | 可作为底仓，仓位可较高 |
| **加仓条件** | 股息率提升或估值进一步下降 |
| **减仓条件** | 分红率下降或财务恶化 |
| **卖出触发** | ① 估值修复完成（股息率降至目标以下）② 分红政策恶化 ③ 财务安全性下降 |
| **止损规则** | 财务恶化即止损，不看价格波动 |

**关键诊断点**：
- 如果用户因"价格下跌"卖出红利股 → **规则冲突**，红利逻辑收益来源是分红，价格下跌股息率上升应持有或加仓

---

### 3. 困境反转逻辑

| 阶段 | 操作规则 |
|------|----------|
| **买入** | 困境底部确认 + 反转催化剂可见 + 财务安全 |
| **仓位** | 试探仓位5-10%，反转信号确认后加仓 |
| **加仓条件** | 反转信号出现（业绩拐点、行业数据改善） |
| **减仓条件** | 反转进程受阻或放缓 |
| **卖出触发** | ① 反转逻辑兑现 ② 反转失败确认 ③ 时间超预期无进展 |
| **止损规则** | 反转失败确认即止损；若底部判断失误，可设成本线止损兜底 |

**关键诊断点**：
- 如果用户在反转验证过程中因价格波动卖出 → **规则冲突**，困境反转需要时间验证，不应因短期波动退出

---

### 4. 事件驱动逻辑

| 阶段 | 操作规则 |
|------|----------|
| **买入** | 事件确定性高 + 时间窗口明确 + 预期未被透支 |
| **仓位** | 固定仓位，不加仓 |
| **卖出触发** | ① 事件兑现当日卖出 ② 事件取消/延期 ③ 时间窗口结束未兑现 |
| **止损规则** | 事件取消即止损；成本线止损兜底 |

## 体系优化机制

当用户积累5次以上同类型复盘后，可发起体系审视：

**触发指令**：用户说"帮我审视一下交易体系"或"优化一下操作策略"

**审视内容**：
1. 统计各类逻辑的交易次数、成功率、平均收益率
2. 识别高频问题模式
3. 分析问题原因
4. 提出改进建议
5. 更新操作模板

## 示例：诊断对话

**用户**：巨化股份，景气度逻辑，买入逻辑是制冷剂涨价确定，25年10月买入，26年4月跌到成本线清仓，但我判断逻辑还在成立。

**AI诊断**：
> **诊断结果**：
> 你这次交易的核心问题是：**理论规则与实际操作冲突**。
>
> 你判断"景气逻辑仍在成立"，却因"跌到成本线"清仓。根据景气度逻辑规则，应该"逻辑止损"而非"成本线止损"。
>
> **问题根源**：
> "不亏本"的情绪底线压倒了理性判断。浮盈回撤触发恐慌，导致在没有逻辑失效信号的情况下卖出。
>
> **改进方案**：
> 1. **心态层面**：设置心理止损线（买入价-5%）替代成本线，给自己缓冲空间
> 2. **仓位层面**：降低初始仓位至5%，减轻心态压力
> 3. **持仓自检**：每周问"逻辑是否成立？"，成立则不看价格波动
>
> 你觉得这些改进建议可以采纳吗？
