# Notes Hunting Guide

## 适用场景

当用户希望你继续在财报里找答案，或者你已经发现异常、需要回附注定位证据时，读取本文件。

## 核心目标

附注追踪不是为了把附注复述一遍，而是为了回答：

- 异常背后的证据在哪里
- 哪些解释能被财报支持
- 哪些地方财报仍然没有讲清楚

## 默认追踪顺序

### 1. 先看主表，确定疑点属于哪一类

常见疑点分类：

- 收入质量问题
- 回款问题
- 存货风险
- 利润质量问题
- 预付款项和供应链问题
- 合同负债和订单质量问题
- 减值风险

### 2. 再按疑点类型回查附注

#### 收入质量相关

优先回查：

- 分产品收入
- 分地区收入
- 收入确认政策
- 前五大客户
- 合同资产
- 应收账款账龄

重点想回答：

- 收入主要来自哪里
- 是否依赖少数客户
- 收入确认口径有没有明显变化

#### 回款相关

优先回查：

- 应收账款账龄
- 坏账准备政策
- 应收票据
- 合同资产

重点想回答：

- 钱为什么没回来
- 回款慢是行业特征还是客户恶化
- 坏账风险有没有提前暴露

#### 存货相关

优先回查：

- 存货构成
- 跌价准备
- 库龄或周转信息

重点想回答：

- 存货堆在哪一类
- 是原材料、在产品还是库存商品出了问题
- 是否已经开始减值

#### 合同负债相关

优先回查：

- 合同负债附注
- 预收性质说明
- 重大订单或交付安排
- 管理层讨论与分析中的订单、交付、产能说明

重点想回答：

- 是客户先付款，还是业务模式天然如此
- 后续能否顺利转成收入
- 是否存在低价锁单或交付压力

#### 预付款项相关

优先回查：

- 预付款项附注
- 主要预付款对象
- 采购安排
- 上游原材料说明

重点想回答：

- 公司为什么要先给钱
- 是锁原材料，还是议价能力弱
- 预付款是否异常集中

#### 利润质量相关

优先回查：

- 非经常性损益明细
- 投资收益
- 公允价值变动收益
- 资产处置收益
- 政府补助
- 资产减值和信用减值

重点想回答：

- 利润是不是靠主业赚出来的
- 利润有没有一次性因素撑着

### 3. 再看管理层讨论与分析

管理层讨论与分析适合回答：

- 为什么收入变化
- 为什么毛利率变化
- 为什么期间费用变化
- 为什么现金流变化
- 为什么存货或应收变化

使用原则：

- 把管理层解释当作线索，不直接当作事实结论
- 必须用主表或附注去核对

## 常见追踪模板

### 模板一：收入增长但现金流弱

回查顺序：

1. 应收账款账龄
2. 合同资产
3. 存货构成
4. 销售回款
5. 管理层对账期或客户变化的解释

### 模板二：合同负债增加但毛利率下降

回查顺序：

1. 合同负债附注
2. 分产品毛利率
3. 收入结构变化
4. 重大订单说明
5. 管理层对价格、成本、交付的解释

### 模板三：净利润很好但扣非一般

回查顺序：

1. 非经常性损益明细
2. 政府补助
3. 投资收益
4. 资产处置收益
5. 公允价值变动

## 输出建议

当需要把用户引回财报里自己看时，优先按下面结构输出：

```text
【附注追踪清单】
- 当前疑点：
- 先看哪张表：
- 再看哪条附注：
- 管理层讨论与分析应看什么：
- 想验证的问题：
- 看完后哪几种结果分别意味着什么：
```

## 使用提醒

- 不要泛泛地让用户“去看附注”，要明确到附注类型和目的
- 如果财报没披露到足够细，直接说明“财报内暂不能确认”
- 附注和管理层解释互相矛盾时，优先相信主表和可量化附注
